Journal Mentor — User Manual

Dialectic System v3.1

How It Works

You write a journal entry. An AI mentor reads it and tells you what you can't see about yourself. You can then engage in dialogue — push back, go deeper, correct the mentor. The system tracks your psychological patterns ("bugs"), goals, and priors over time, getting sharper with every entry.

Everything runs locally on your Mac. The only external call is to the Anthropic API for analysis.

Journal Tab

Writing: Choose Free-form or Guided mode. Add an optional title. Write your entry and press Submit (or Cmd+Enter). The mentor's analysis appears on the right.

Dialectic: After the analysis, use the text input at the bottom to respond. Push back, ask for clarity, go deeper. The mentor adjusts.

History: Click any entry in the list below the writing area to review it and its analysis. Click "+ New Entry" to return to writing mode.

Backup: Click the Backup button (top right) to manually push the current entry to your Google Sheet.

What gets tracked per entry: Tags, emotional valence (-2 to +2), commitments made, processing mode (abstract vs concrete), somatic content, schema level, insight-to-action ratio, containment level, defense mechanisms, relational patterns (CCRT).

Debugging Matrix Tab

Your recurring psychological patterns, tracked as "bugs." Each bug has a fire count (how often it appears in your entries), status (active/declining/resolved), and optionally an intervention protocol.

Adding patterns: Click "Add Pattern" and fill in the name + description. Expand "+ Intervention fields" to add trigger, thought signature, behavior signature, and intervention protocol.

Mentor-proposed bugs: When the mentor detects a new pattern, it appears as a pending bug (amber). Confirm or Dismiss it.

Intervention protocols: Click "Intervention Protocol" on any bug to see its trigger → thought → behavior → intervention chain. The mentor uses these to guide its response when the pattern fires.

Mentor Log Tab

Full transparency of everything the system does internally. Every bug fire, memory update, experiment, prior change, and safety flag is logged here.

Filtering: Use the filter buttons to see specific categories (Memory, Bugs, Priors, Experiments, Safety, Corrections).

Responding: Click "Respond" on any log entry to correct the mentor. Your corrections are weighted heavily in future memory synthesis.

Pending items: The badge on the tab shows how many items need your review (proposed bugs, audit recommendations).

System Tab

Goals: Two columns — things you're moving toward and things you're moving away from. The mentor references these in every analysis.

Priors: Key beliefs the system tracks about you (e.g., "Commitment follow-through rate: 20-25%"). Click "Seed Initial Priors" after uploading a baseline. Each prior accumulates evidence over time and can be manually adjusted.

Phases: Track developmental phases (e.g., "Exploration", "Integration"). Adding a new phase closes the previous one.

Reports: Generate weekly accountability reports or monthly deep reads + system audits. Reports are archived and can be emailed.

Documents: Upload supporting documents. Types matter:

  • Foundational Research (blue) — explains WHY patterns exist. Mark as "included in baseline" after uploading.
  • Practical Playbook (green) — describes WHAT TO DO when patterns fire. Click "Decompose" to extract structured bugs with intervention protocols.
  • Narrative Analysis (purple) — identity/meaning analyses. Feed baseline evolution.

Baseline: Your psychological profile. Upload a file or paste text. This is the foundation the mentor reads before every entry. It gets versioned — old versions archived automatically.

Micro-Assessments: Quick daily check-ins (wellbeing, interpersonal, social role, overall + body sensation). The mentor uses these to detect deterioration trends.

Show Me Your Model: Generates a plain-language summary of what the system currently believes about you. Read it and correct anything wrong.

How the System Gets Smarter

Every entry: Memory note saved, bugs checked, metadata captured, experiments tracked.

Every 10 entries: Raw memory notes are compressed into a sharper model. Your corrections are weighted heavily.

Weekly: Accountability report with goal adherence, commitment tracking, experiment review, bug analysis.

Monthly: Deep read + full system audit. Bug promotions/demotions, prior updates, baseline evolution, phase assessment — all staged for your approval.

Safety Features

Crisis detection: If the mentor detects crisis language, it stops all analysis and responds with support + crisis resources.

Containment: If you're in acute distress, the mentor shifts to supportive mode — no deep pattern analysis, just acknowledgment.

Rumination detection: 3+ consecutive abstract-evaluative entries trigger a redirect to behavioral experiments.

Deterioration detection: Declining trends across entries trigger a check-in and may recommend professional support.

Setup Checklist

  1. Set ANTHROPIC_JOURNAL_KEY in ~/.zshrc
  2. Upload your baseline profile (System tab → Baseline)
  3. Click "Seed Initial Priors" (System tab → Priors)
  4. Upload practical playbooks and click "Decompose" to populate the debugging matrix
  5. Set your goals (System tab → Goals)
  6. Optional: Set JOURNAL_GDOC_WEBHOOK for Google Sheet backup
  7. Optional: Set JOURNAL_EMAIL_WEBHOOK for report emails
All data stays on your Mac. Back up by copying the ~/journal-mentor folder.
